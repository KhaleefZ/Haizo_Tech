Our works

What we've built, and what it changed

Systems running in production. Each one says what was wrong before, what we actually did, and what changed afterwards — measured where we can measure it and described plainly where we can't.

These filters are admin-driven. They render from GET /v1/work-categories sorted by order, which is exactly what the Categories manager in the admin writes. Add, rename or reorder a category there and this row changes with it — no deploy. Each card's category is a foreign key to the same table, so a rename can never leave a card orphaned or a filter matching nothing.
